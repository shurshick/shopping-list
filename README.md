# Список покупок

Мобильное приложение для Android и сервер синхронизации для собственного TrueNAS/Docker-сервера.

## Что умеет проект

- регистрация и вход пользователей;
- несколько списков покупок;
- отметка товаров как купленных;
- общий доступ к списку по email другого пользователя;
- синхронизация при запуске приложения и после изменений;
- первичная настройка сервера через веб-мастер;
- разворачивание серверной части из исходников или готового образа GitHub Container Registry.

## Состав проекта

- `backend` - серверная часть на FastAPI.
- `android` - Android-приложение на Kotlin.
- `docker-compose.yml` - запуск сервера со сборкой backend на месте.
- `docker-compose.github-build.yml` - запуск сервера со сборкой backend напрямую из публичного GitHub-репозитория.
- `docker-compose.ghcr.yml` - запуск готового backend-образа из GitHub Container Registry.
- `docs` - инструкции по API, TrueNAS и GitHub-развертыванию.

## Быстрый запуск сервера

1. Скопируйте `.env.example` в `.env`.
2. Замените `POSTGRES_PASSWORD` и `JWT_SECRET` на длинные случайные значения.
3. Запустите:

```bash
docker compose up -d --build
```

Для временного публичного теста без токенов используйте `docker-compose.github-build.yml`. Подробная инструкция: `docs/github-deploy.md`.

Проверка:

```bash
curl http://localhost:8000/health
```

## Первичная настройка

После запуска Docker откройте:

```text
http://truenas-ip:8000/setup
```

При первом запуске мастер попросит создать администратора: email и пароль. В этой же форме укажите внешний HTTPS-адрес сервера, название приложения и параметры регистрации.

Позже настройки можно менять на этой же странице `/setup`, введя email и пароль администратора.

## Доступ из интернета

Для доступа из интернета используйте доменное имя и HTTPS через reverse proxy на TrueNAS, например Nginx Proxy Manager, Caddy или Traefik. Публиковать API напрямую без HTTPS не стоит, потому что приложение передает логин, пароль и токен авторизации.

## Android-приложение

При первом входе в приложении укажите адрес сервера, например `https://shopping.example.com`. Адрес сохраняется на телефоне и используется для регистрации, входа и синхронизации.
