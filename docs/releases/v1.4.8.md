# Список покупок v1.4.8

Релиз Android-диагностики подключения и синхронизации.

## Что изменилось в этом релизе

- В настройках Android добавлена кнопка `Диагностика`, без перегрузки основного меню приложения.
- Экран диагностики показывает текущий сервер, тип сервера, версию backend и публичное состояние `/health/live`, `/health/ready`, `/server-config`.
- Добавлены сведения о синхронизации: последняя успешная синхронизация, последняя попытка, последняя ошибка и количество ожидающих offline operations.
- Добавлены действия `Проверить подключение`, `Повторить синхронизацию`, `Скопировать диагностику` и безопасная очистка локального кэша с подтверждением.
- Диагностический отчёт не содержит token, password, Authorization header, email, товары, списки и payload offline queue.
- Версии обновлены до `API 1.4.8` и `Android 1.4.8` (`versionCode = 31`).

## Совместимость

- Android: 8.0 Oreo и новее, `minSdk = 26`.
- Docker image: `ghcr.io/shurshick/shopping-list-api:latest` и `ghcr.io/shurshick/shopping-list-api:v1.4.8`.

## Файлы релиза

- `shopping-list-android-v1.4.8.apk`
- `shopping-list-server-v1.4.8.zip`
- `shopping-list-source-v1.4.8.zip`

## Обновление сервера

1. Сделайте backup БД или выполните `python -m app.cli backup --output backup.json`.
2. Обновите Docker image до `latest` или `v1.4.8`.
3. Перезапустите контейнер API.
4. Проверьте `/health/ready` и `/admin/system`.
